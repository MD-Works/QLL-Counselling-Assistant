Quick start
Minimum required — paste before </body>
<script src="https://qll-assistant.pages.dev/widget.js" data-groq-key="gsk_YOUR_KEY_HERE" ></script>
Note: The widget loads the knowledge base from the same domain by default. If the widget.js and qll-knowledge-base.json are not on the same server, use data-kb-url to point to the correct URL.
All configuration attributes
AttributeDefaultOptions / notes
data-groq-key required Your Groq API key. Free at console.groq.com
data-bot-name "Lea" Display name in header and messages
data-personality "warm" warm · professional · bright · grounded
data-avatar "🌿" Any single emoji or short text (e.g. "Q", "✦")
data-accent-color "#4a9b8e" Hex colour for bubble, header highlights, send button
data-accent-dark "#2d6b61" Darker shade used for the header background
data-bg-color "#1e2a3a" Chat panel background colour
data-position "right" right · left — which corner the widget sits in
data-greeting auto Override the opening message. Leave blank to use the default.
data-groq-model "llama-3.1-8b-instant" llama-3.1-8b-instant · llama-3.3-70b-versatile · gemma2-9b-it
data-kb-url "qll-knowledge-base.json" URL to the knowledge base JSON file
data-debug "false" true · false — shows model/token/source info in a bar below the header
Personality presets
Pick one — controls the bot's tone and response style
Warm & Nurturing
data-personality="warm"
Gentle, empathetic, reassuring. Acknowledges feelings before giving information. Best for clients who may be anxious or vulnerable.
Calm & Professional
data-personality="professional"
Measured, clear, trustworthy. Like a knowledgeable receptionist. Best for practices that want a more formal tone.
Bright & Approachable
data-personality="bright"
Friendly, light, easy to talk to. Accessible everyday language. Best for a welcoming, non-intimidating first impression.
Grounded & Wise
data-personality="grounded"
Thoughtful, unhurried, quietly confident. Considered responses. Best for practices with a holistic or mindfulness focus.

Security note
About the Groq API key

Placing the API key in a data-* attribute means it is visible in the page source. For a low-traffic practice like QLL this is acceptable — the Groq free tier has generous rate limits that make abuse unlikely.

When you are ready to secure the key properly, the solution is a Cloudflare Worker proxy: the widget calls the Worker, the Worker holds the key server-side and calls Groq, and the key is never in the browser. This is Phase 2 of the project roadmap.
