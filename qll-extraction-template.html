Step 1 · The prompt to paste into your other AI chat

Copy this entire prompt — paste it first, before any documents
You are a structured data extraction assistant. Your only job is to read the documents I paste and convert their information into a strict JSON format for a RAG chatbot knowledge base. CRITICAL RULES — follow without exception: 1. Output ONLY valid JSON — no explanations, no commentary, no markdown code fences. 2. Every entry MUST have all required fields (id, category, keywords, question, answer). 3. Write questions exactly as a real client would ask them — natural, conversational. 4. Write answers in third person ("Elizabeth offers..." not "I offer..."). 5. Keep answers factual — do not infer, embellish, or add information not in the document. 6. If information is missing or unclear, use the placeholder: "Please contact Elizabeth directly to confirm." 7. Never include pricing figures unless explicitly stated in the document. 8. Never include specific dates, times, or personal contact details in answer fields. 9. Each entry covers ONE topic only — split multi-topic paragraphs into separate entries. 10. Keywords must be lowercase, single words or short phrases a client might actually type. OUTPUT FORMAT — a single JSON array, nothing else: [ { "id": "unique_lowercase_slug_with_underscores", "category": "services|pricing|booking|credentials|logistics|emergency", "keywords": ["keyword1", "keyword2", "keyword3"], "question": "Natural question a client would ask?", "answer": "Factual answer in third person, 2-4 sentences." } ] I will now paste Elizabeth's documents. Extract every distinct piece of client-relevant information into separate entries. Begin outputting JSON immediately after the last document.
Step 2 · Field definitions & rules
2
Every entry must have these exact fields
Field Required Rules Example
id yes Unique, lowercase, underscores only. Describes the topic in 2–4 words. "service_reiki"
"booking_cancellation"
"pricing_session_fee"
category yes Must be exactly one of the six allowed values below. Pick the closest fit.
services pricing booking credentials logistics emergency
keywords yes Array of 4–10 strings. Lowercase. Include synonyms and alternative phrasings a client might type. Think: what words would someone use when they don't know the formal term? ["reiki", "energy healing", "hands on", "relax", "chakra", "balance"]
question yes A single, natural question a real client would ask. Conversational tone. Ends with a question mark. "What is Reiki and how does it work?"
answer yes 2–4 sentences. Third person ("Elizabeth offers…"). Factual only — no opinions, no promises. If a detail is missing, end with: "Please contact Elizabeth directly to confirm." "Reiki is a gentle energy healing practice… Please contact Elizabeth directly to confirm availability."

Step 3 · Quality checklist — review before accepting output
3
Check the extracted JSON against these rules before pasting into the knowledge base
  • ✓ Every entry has all 5 fields: id, category, keywords, question, answer
  • ✓ All IDs are unique — no two entries share the same id
  • ✓ Category values are exactly one of the 6 allowed strings (no typos, no new categories)
  • ✓ Keywords array has at least 4 items per entry
  • ✓ Questions are phrased as a client would ask, not as a document heading
  • ✓ Answers are in third person — no "I", "we", or "my" referring to the practice
  • ✓ Answers contain no invented information — only what was in Elizabeth's documents
  • ✗ Do NOT include specific phone numbers, email addresses, or street addresses in answers — these change and belong in a separate contact field
  • ✗ Do NOT include pricing figures unless Elizabeth has explicitly confirmed them as current and final
  • ✗ Do NOT merge two topics into one entry — split them
